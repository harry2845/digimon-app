# 数码宝贝电子图鉴

数码宝贝物语 网路侦探 骇客追忆（Digimon Story: Cyber Sleuth - Hacker's Memory）的 Web 版电子图鉴工具。

## 功能

- **图鉴列表**：按编号排序，支持按阶级（幼年期～超究极体）筛选，支持按收集状态（未见过/仅已见过/已拥有/已见过+已拥有）筛选，两种筛选可叠加
- **详情页**：查看中英文名、阶级、进化/退化目标（可点击跳转），左右滑动切换角色
- **编辑模式**：修改名字/阶级、添加/删除进化退化关系、上下移动排序、交换图鉴编号、新增/删除角色
- **收集状态**：每个数码宝贝有三种状态（未见过/已见过/已拥有），列表页和详情页均可点击切换
- **简繁切换**：导航栏一键切换繁体/简体中文显示，搜索支持简体输入，语言偏好自动保存（不影响底层数据）
- **进化路线查询**：选择起点和终点，BFS 算法计算最短进化/退化路径，同时显示：
  - **理想路线**：不受限的最短路径
  - **当前可行路线**：退化只能经过已见过/已拥有的数码宝贝（进化不受限）
  - **途经点**：可添加多个必经中间节点，自动优化经过顺序，寻找最短路径，途经点在结果中以橙色高亮标记
  - **多标签（Tab）**：支持多个 Tab 并行查看不同路线（最多 10 个），每个 Tab 独立保存起点/途经点/终点和查询结果缓存
  - **标签预设**：可将当前 Tab 保存为命名预设（最多 10 个），随时加载恢复
  - **持久化**：Tab 状态和预设保存在 localStorage，JSON 导出/导入时一并包含
- **全收集路线规划**：从已拥有的数码宝贝出发，自动规划一条覆盖尽可能多未拥有数码宝贝的进化/退化链路，列出无法到达的数码宝贝
- **数据管理**：导出/导入 JSON（含收集状态）、保存为默认数据、重置为默认或出厂数据

## 使用方法

浏览器直接打开 `index.html`，无需服务器。

## 文件说明

| 文件 | 说明 |
|------|------|
| `index.html` | 页面入口 |
| `style.css` | 样式（响应式，移动端适配） |
| `app.js` | 应用逻辑（数据层、列表、详情、编辑、收集状态） |
| `pathfinder.js` | BFS 最短路径算法（含受限路径、途经点路径、全收集路线） |
| `data.js` | 当前默认数据（可通过"保存为默认数据"更新） |
| `data_backup.js` | 出厂数据备份（不可修改） |
| `build_data.py` | 数据生成脚本：从 `digimon_guide.txt` 生成 `data.js` |

## 数据说明

- 共 341 个数码宝贝，数据来源为游戏内部数据文件 + Wikimon 中英文名对照
- 阶级包括：幼年期I、幼年期II、成長期、成熟期、完全體、究極體、超究極體、裝甲體
- 每个角色有唯一 ID（uid），图鉴编号（dexId）可通过编辑模式交换调整
- 部分中文名可能与游戏内实际名称有出入，可在编辑模式中修正
- 图鉴数据修改存储在浏览器 localStorage 中，刷新不会丢失
- 收集状态单独存储在 localStorage，导出 JSON 时会一起导出
